🎨 OpenCV Image Cartoonizer
본 프로젝트는 OpenCV 라이브러리를 활용하여 일반 사진을 만화(Cartoon) 스타일로 변환하는 이미지 처리 프로그램입니다. Bilateral Filter와 Adaptive Thresholding 기술을 조합하여 수채화 같은 색감과 뚜렷한 외곽선을 구현했습니다.

🚀 기능 설명 및 실행 화면
1. 사용된 주요 기술
Median Blur: 이미지의 노이즈를 제거하여 깨끗한 외곽선을 추출할 수 있게 합니다.

Adaptive Thresholding: 조명 변화에 유연하게 대응하여 물체의 테두리(Edge)를 생성합니다.

Bilateral Filter: 경계선을 유지하면서 내부 색상만 부드럽게 뭉쳐 만화 같은 질감을 표현합니다.

2. 데모 (Demo)
✅ 만화 느낌이 잘 표현되는 예시 (Success Case)
분석: 피규어와 같이 배경이 단순하고 색 대비가 뚜렷한 이미지에서는 외곽선이 깔끔하게 추출되며, Bilateral Filter를 통한 색상 단순화 효과가 극대화됩니다.

❌ 만화 느낌이 잘 표현되지 않는 예시 (Failure Case)
분석: 질감이 복잡하거나 미세한 디테일이 너무 많은 영역(예: 복잡한 배경, 머리카락 등)에서는 Adaptive Threshold가 과도한 검은 선을 생성하여 화면이 지저분해 보이는 현상이 발생합니다.

🧐 알고리즘 분석 및 한계점
1. 장점 (Pros)
엣지 보존: 단순한 가우시안 블러와 달리 물체의 형태(Edge)를 파괴하지 않고 내부 색상만 부드럽게 변환합니다.

조명 적응성: 이미지의 국소적인 밝기 변화에 따라 임계값을 조절하는 Adaptive Threshold를 사용하여 다양한 환경의 사진에 대응할 수 있습니다.

2. 한계점 (Limitations) - [평가 항목 5점]
파라미터 수동 조절: 본 알고리즘의 수치(9, 5 등)는 특정 해상도에 최적화되어 있습니다. 이미지의 해상도가 너무 높거나 낮으면 외곽선이 너무 얇거나 두껍게 표현되는 해상도 의존성 문제가 있습니다.

복잡한 배경 노이즈: 숲이나 복잡한 패턴의 옷과 같이 미세한 선이 많은 경우, 이를 모두 '외곽선'으로 인식하여 결과물이 지저분해지는 한계가 있습니다.

색상 단계의 부재: 실제 만화는 색상을 몇 가지 단계로 끊어서 표현(Posterization)하지만, 본 알고리즘은 단순히 부드럽게 뭉치는 수준에 그쳐 전문적인 카툰 느낌에는 미치지 못합니다.
